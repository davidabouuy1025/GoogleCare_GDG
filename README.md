# GoogleCare_GDG

## Run Locally

**Prerequisites:**  Node.js

1. Install dependencies:
   `npm install`
2. Set the `GEMINI_API_KEY` in [.env.local](.env.local) to your Gemini API key
3. Run the app:
   `npm run dev`

## What does each file do
### Core Configuration Files
1. package.json: The heart of the project. It defines project metadata, external libraries (dependencies like firebase, express, @google/genai), and execution scripts. The dev script is configured to run the custom server.ts using tsx.

2. tsconfig.json: Configures the TypeScript compiler. It defines how TypeScript code is transformed into JavaScript, setting the target version to ES2022 and enabling modern features like react-jsx.

3. vite.config.ts: The configuration for the Vite build tool. It integrates React and Tailwind CSS plugins, handles environment variables (like your Gemini API Key), and manages path aliases (using @ to refer to the root).

4. metadata.json: A platform-specific file for AI Studio. It defines the app's name, description, and explicitly requests browser permissions for the camera, microphone, and geolocation.

5. .env.example: A template file that lists the environment variables required for the app to function (e.g., GEMINI_API_KEY) without exposing actual secrets.

6. .gitignore: Tells Git which files and directories to ignore (like node_modules or build artifacts) to keep the repository clean and secure.
### Backend & Infrastructure
1. **server.ts**: The entry point for the Node.js environment. It uses Express to create a web server. In development, it integrates Vite as middleware to provide fast updates; in production, it serves the compiled static files from the dist folder.
2. **firestore.rules**: The security layer for your database. It defines who can read or write data. It uses custom logic to ensure patients can only see their own records while allowing your specific email (myleejm23@gmail.com) to act as an administrator.

3. **firebase-blueprint.json**: A structural map of your database. It defines the "shape" of your data (the User, Symptom, and Mood entities) and where they live in Firestore. It acts as a reference for the AI and developers to maintain data consistency.

4. **firebase-applet-config.json**: Contains the unique credentials (API Key, Project ID, App ID) generated by Google Cloud to connect your specific application to the Firebase backend services.
### Frontend Source Code (/src)
1. **src/main.tsx**: The JavaScript entry point. It initializes the React application and mounts the root App component into the index.html DOM.

2. **src/App.tsx**: The primary application file. It contains:
   -  **UI Logic**: Manages the navigation tabs (Dashboard, Symptoms, Wound, etc.).
   - **State Management**: Tracks the current user and patient data.
   - **Firebase Integration**: Uses onSnapshot for real-time database updates and signInWithPopup for Google Authentication.
   - **Business Logic**: Handles the logic for emergency triggers and profile updates.

3. **src/firebase.ts**: Initializes the Firebase SDK. It exports the db (Firestore) and auth (Authentication) instances used throughout the app, ensuring they use the correct configuration from firebase-applet-config.json.

4. **src/services/aiService.ts**: The AI integration layer. It uses the Google GenAI SDK to communicate with Gemini models. It contains specific functions to analyze medical symptoms, evaluate elderly check-ins, and process wound images.

5. **src/index.css**: The global stylesheet. It uses @import "tailwindcss" to inject utility classes into the project, enabling the modern, responsive design seen in the UI.

### Miscellaneous
1. **index.html**: The skeleton of the web application. It provides the <div id="root"> where React renders the UI and loads the main.tsx script.

2. **wound_model.py**: A reference Python script. While the live app uses Gemini Vision for wound analysis, this file serves as a blueprint for how a custom TensorFlow/Keras model could be trained for the same purpose.
